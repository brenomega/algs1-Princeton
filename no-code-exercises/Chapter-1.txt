Social network connectivity. 

Given a social network containing n members and a log file containing m timestamps at 
which times pairs of members formed friendships, design an algorithm to determine the earliest time at which all members 
are connected (i.e., every member is a friend of a friend of a friend ... of a friend). Assume that the log file is sorted by 
timestamp and that friendship is an equivalence relation. The running time of your algorithm should be m log n or better and 
use extra space proportional to n.  

<<<SOLUTION>>>

Analogy: Each member is a vertex in a graph, and each log entry represents a union operation. In this way, it is enough to 
monitor at which log entry the vertices become fully connected.  

Algorithm: We will use weighted quick-union with path compression to implement the problem's structure. Given the implementation 
already seen in the course, it is sufficient to create an "allConnected()" method and call it whenever "union()" is invoked:  

1. Create a private variable "int rm;" and initialize it in the constructor as "rm = N;".  
2. Update the "union()" method with "rm--;" whenever a union is actually performed, i.e., after "if (pRoot == qRoot) return;".  
3. Create the method "allConnected()" with "return rm == 1;".  
4. Call "allConnected()" every time "union()" is called, either internally in the algorithm or externally in the application.  

After this, you can use the system structure to retrieve the timestamp of the last log entry.

Union-Find with Specific Canonical Element.

Add a method "find()" to the union-find data type so that "find(i)" returns the largest element in the connected component 
containing "i". The operations "union()", "connected()", and "find()" should all take logarithmic time or better.

For example, if one of the connected components is {1, 2, 6, 9}, then the "find()" method should return 9 for each of the 
four elements in this connected component.

<<<SOLUTION>>>

Algorithm: The idea is to maintain a new array that stores the largest element in each tree's root, updating it whenever a union occurs.

1. Declare and initialize the array:
   private int[] bg;
   // in the constructor:
   bg = new int[N];
   for (int i = 0; i < N; i++) {
       bg[i] = i;
   }

2. Update union(): insert the following immediately after:
   if (pRoot == qRoot) return;
   // then add:
   if (bg[pRoot] < bg[qRoot]) {
       bg[pRoot] = bg[qRoot];
   } else {
       bg[qRoot] = bg[pRoot];
   }

3. Create "find()" with signature and body:
   int find(int i) {
       int iRoot = root(i);
       return bg[iRoot];
   }

Design a data type so that all operations (except construction) take logarithmic time or better in the worst case.

<<<SOLUTION>>>

The base logic will be very similar to the previous problem, so we will use it as a foundation.

1. Given the weighted quick-union implementation with path compression seen in the course, create the arrays:
   - "private int[] max;"
   - "private boolean[] removed;"
   Initialize "max" exactly the same way as "id", and "removed" as "removed = new boolean[N];".

2. Mark the "union()" method as private and remove the "connected()" method, since they are not intended for external 
use in our API.

3. Do the same as step 2 from the previous exercise, but this time using the "max" array instead of "bg".

4. Create the method "successor()" with the signature and body:

   public int successor(int i) {
       if (!removed[i]) return i;
       int successor = max[root(i)] + 1;
       if (successor < id.length) return successor;
       return -1;
   }

Search in a Bitonic Array.

An array is bitonic if it is comprised of an increasing sequence of integers followed immediately by a decreasing 
sequence of integers. Write a program that, given a bitonic array of n distinct integer values, determines whether 
a given integer is in the array.

Standard version: Use ~3 log n comparisons in the worst case.

Signing bonus: Use ~2 log n comparisons in the worst case (and prove that no algorithm can guarantee to perform 
fewer than ~2 log n comparisons in the worst case).

<<<SOLUTION>>>

1. Find the "peak" using a specialized binary search:
   1. lo = 0 (lower bound), hi = nums.length - 1 (upper bound)
   2. while lo < hi:
       i = (lo + hi) / 2
       if nums[i] < nums[i+1] then lo = i + 1
       else hi = i
   3. Return lo as the "peak"

2. Perform binary search in parallel on both halves:
   1. lo1 = 0, hi1 = peak; lo2 = peak, hi2 = nums.length - 1
   2. while lo1 <= hi1 OR lo2 <= hi2:
       1. If lo1 <= hi1:
           - i1 = (lo1 + hi1) / 2
           - if nums[i1] > n then hi1 = i1 - 1
           - else if nums[i1] < n then lo1 = i1 + 1
           - else return TRUE
       2. If lo2 <= hi2:
           - i2 = (lo2 + hi2) / 2
           - if nums[i2] > n then lo2 = i2 + 1
           - else if nums[i2] < n then hi2 = i2 - 1
           - else return TRUE

Return FALSE if the element is not found in either half.

Egg Drop Problem.

Suppose that you have an n-story building (with floors 1 through n) and plenty of eggs. An egg breaks if it is 
dropped from floor T or higher and does not break otherwise. Your goal is to devise a strategy to determine the 
value of T given the following limitations on the number of eggs and tosses:

Version 0: 1 egg, <= T tosses.

Version 1: ~1 log n eggs and ~1 log n tosses.

Version 2: ~log T eggs and ~2 log T tosses.

Version 3: 2 eggs and ~2 sqrt(n) tosses.

Version 4: 2 eggs and <= c sqrt(T) tosses for some fixed constant c.

<<<SOLUTION>>>

Represent the building as an array of 0's and 1's where 1 indicates a floor where the egg breaks (ER(0*1*)).

0. Linear Search:
   - Check each floor sequentially from 1 to n until the egg breaks.
   - Tosses required: T (worst case).

1. Binary Search:
   - Apply standard binary search to find T.
   - Tosses required: ~log n (worst case).

2. Exponential Jumps:
   - Jump floors exponentially: 1, 2, 4, 8, ... until an egg breaks.
   - Perform binary search within the last jump interval.
   - Tosses required: ~log T (jump) + log T (binary search) = ~2 log T.

3. Fixed Block Size:
   - Divide floors into blocks of size √n.
   - Check each block sequentially until the egg breaks.
   - Perform linear search within the target block.
   - Tosses required: √n + √n = ~2√n.

4. Increasing Block Sizes:
   - Use blocks of increasing size: 1, 2, 3, ..., k.
   - Find the block where the egg breaks and search linearly within it.
   - Tosses required in worst case: c√T, where c ≈ 2√2.

Hot or Cold Number Guessing Game.

Goal: Guess a secret integer between 1 and N. After each guess:

- If the guess equals the secret number, the game stops.  
- Otherwise (starting with the second guess), you learn whether the guess is hotter (closer to) or colder
(farther from) the secret number compared to your previous guess.

Design an algorithm that finds the secret number in ~1 log N guesses.  
- Hint: First design a solution assuming guesses can be made in the extended range [-N, 2N], then adapt it
to the original range [1, N].

<<<SOLUTION>>>

Idea: Mirror the previous guess around the midpoint to determine which half contains the secret number.

1. Initialize the previous guess: g_prev = 0  
2. Initialize the search interval: L = 0, R = n  
3. While L < R do:  
   - m = (L + R) / 2  
   - g = 2 * m - g_prev  (mirror around midpoint)  
   - If guess "equals" secret → return g  
   - If guess is "hotter" than previous:  
       - If g_prev ≤ m → L = m + 1  
       - Else → R = m  
   - If guess is "colder" than previous:  
       - If g_prev ≤ m → R = m  
       - Else → L = m + 1  
   - Else → return m  
4. Return L

Time complexity: ~log n

<<<SOLUTION>>>

Stack with max. 

Create a data structure that efficiently supports the stack operations (push and pop) and also a 
return-the-maximum operation. Assume the elements are real numbers so that you can compare them.

<<<SOLUTION>>>

Base: two control structures (main stack and auxiliary stack)

Push(x): * Insert the element into the main stack.
If the auxiliary stack is empty or if the element is greater than or equal to the current top of the auxiliary stack, insert it into the auxiliary stack as well.
Pop(): * Remove the element from the main stack.
If the removed value is equal to the top of the auxiliary stack, remove it from the auxiliary stack as well.
GetMax(): * Simply consult the top of the auxiliary stack without removing it.
